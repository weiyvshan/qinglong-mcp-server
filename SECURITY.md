## 报告安全漏洞

**请不要**在公共 issue 中报告安全漏洞。

如果你发现了一个安全漏洞，请通过以下方式私下报告：

- 发送邮件至：[your-security-email@example.com]

请在报告中包含以下信息：

1. 漏洞的详细描述
2. 复现步骤
3. 受影响的版本
4. 可能的修复建议（如果有）

## 响应流程

1. **确认收到**：我们会在 48 小时内确认收到你的报告
2. **调查评估**：我们会评估漏洞的严重性和影响范围
3. **修复开发**：开发修复补丁
4. **发布更新**：发布安全更新并公开披露（在修复后）

## 安全最佳实践

使用 qinglong-mcp-server 时，请遵循以下安全建议：

### 环境变量安全

- 不要将 `QL_CLIENT_ID` 和 `QL_CLIENT_SECRET` 硬编码在代码中
- 使用环境变量或安全的密钥管理系统
- 定期轮换 Client Secret
- 不要将这些密钥提交到版本控制

### 网络安全

- 确保青龙面板使用 HTTPS
- 限制青龙面板的访问 IP
- 使用强密码保护青龙面板账号

### 权限最小化

- 为 MCP 服务器创建专用的青龙面板应用
- 只授予必要的权限
- 定期审查应用权限
